# crosstab

## 目的（誰の何を解決するか）
不特定多数のユーザー（顧客・第三者）が、2つのカテゴリ変数の関係を**統計的に正しく**判断できるようにするツール。単なるクロス集計表（ピボットテーブル）は「男性の60%、女性の50%が買った」を見せるだけで、それが**意味のある差なのか偶然の偏りなのか**を答えない。本ツールはカイ二乗検定・調整済み残差・効果量（Cramér's V）を自動で計算し、「どのセルが・どれくらい・偶然を超えて」効いているかを客観的に返す。

**このツールは統計処理エンジンであり、「クロス分析」という*思想・方法論*（`methods/cross-analysis/`）とは別物。** 名前が似ているが混同しないこと。思想側は人間 vs AI の領域論、こちらは分割表の統計。参照関係はなく、ファイルも共有しない。

## 技術構成
- 言語: Python 3.9+
- 依存: `scipy`（カイ二乗検定・分布）/ `pandas`（records→分割表・汚いデータの型強制）/ `pydantic`（入出力スキーマの契約）
- 形態: JSON in → JSON out のエンジン。CLI（`python -m crosstab`）とライブラリ（`analyze()`）の両方で呼べる。Web UI や LLM からは JSON 契約経由で接続する（言語非依存）。
- 依存宣言は**このツール直下**（`pyproject.toml`）に置く。リポジトリルートには置かない（モノレポ混在防止・AGENTS.md 全体ルール）。

## 決まったこと
- スタック = Python（scipy/pandas/pydantic）。理由: カイ二乗・残差は検証済み実装（scipy）を使い、統計の正しさを保証する（2026-07-23 ユーザー決定。詳細は `docs/DECISIONS.md`）。
- 3つの柱を層で分離: `validate`（入力の正規化・不正検出）→ `stats`（統計の中核）→ `schema`（入出力契約）。
- **人間チェックポイントを出力に埋め込む**: 研究（`methods/cross-analysis/`）の結論「解釈・結論は人間（B軸）」に従い、出力に「これは統計的示唆であって結論ではない／因果ではない／人間が現場文脈で判断せよ」を必ず含める。
- 統計的誠実さ: p値だけでなく**効果量**と**前提チェック**（期待度数≥5の割合など）を必ず返し、「有意だが些細」「N過小で不安定」を利用者が誤読しないようにする。
