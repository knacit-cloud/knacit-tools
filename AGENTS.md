# knacit-tools — AI エージェント向けルール

このファイルはリポジトリ全体のルール。各ツールの個別ルールは `tools/<ツール名>/AGENTS.md` にあり、そちらが優先される。

## このリポジトリは何か
- [Knacit](https://knacit.com)（BtoB伴走支援）が公開する、誰でも使える便利ツール集。
- 目的は実用と、Knacit の技術力を目に見える形で示すこと。コードだけでなく、設計判断の記録（`docs/DECISIONS.md`）も成果物の一部。

## 絶対ルール（セキュリティ）
- **このリポジトリは公開（Public）**。全ファイルが全世界から見られる前提で書く。
- APIキー・トークン・個人メールアドレス・顧客情報は**いかなるファイルにも書かない**。秘密情報は環境変数（`.env` / `.dev.vars`、gitignore済み）に置き、ファイルには参照方法だけ書く。
- `.claude/settings.local.json` はコミットしない（gitignore済み）。
- コミット前に `git status` で意図しないファイルが混ざっていないか確認する。

## ディレクトリ構成
```
AGENTS.md            ← このファイル（全体ルール）
CLAUDE.md            ← AGENTS.md へのシンボリックリンク
README.md            ← リポジトリの顔（日英併記）
docs/DECISIONS.md    ← 設計判断の記録（なぜそう決めたか）
.claude/
  settings.json      ← 共有設定（コミットする）
  skills/            ← スキル置き場（必要になったら導入）
tools/
  <ツール名>/
    AGENTS.md        ← ツール固有の文脈（目的/構成/決定事項/課題/確認方法）
    README.md        ← ツールの使い方（公開用）
```

## 作業の進め方
- 実装の前に必ず計画を提示し、ユーザーの確認を取ってから着手する。
- 大きな設計判断をしたら `docs/DECISIONS.md` に1行でも記録する。
- ツールを新規作成するときは `tools/<ツール名>/` を作り、AGENTS.md（下記テンプレ）から書き始める。

## ツール用 AGENTS.md テンプレ
```
# <ツール名>
## 目的（誰の何を解決するか）
## 技術構成
## 決まったこと
## 未解決の課題
## 動作確認の方法
```

## 言語ポリシー
- README.md（公開用ドキュメント）: 日英併記。英語は米国綴り（center / organization）。
- AGENTS.md・docs/ など内部向け: 日本語のみ。

## コミット規約
- メッセージは英語の命令形1行（例: `Add markdown table formatter`）。本文が必要なら空行を挟んで日本語可。
- 1コミット = 1つの意味のある変更。
